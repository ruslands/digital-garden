## Revolut Interview Process

### Interview Format

**Company:** Revolut

**Key Requirements:**
- **Unit tests** - Writing and understanding unit tests
- **Mocks** - Using mocks in testing
- **Share screen** - Screen sharing during interview
- **Google Chrome** - Using Chrome browser
- **Shortcuts** - Try using keyboard shortcuts and decorators
- **Gain implicit and explicit requirements** - Understanding both stated and unstated requirements

### Task Structure

- **Initial task:** One task at the beginning
- **Progressive complexity:** More tasks will be added during the test
- **Time limit:** 45 minutes
- **Minimum completion:** At least 3 tasks should be completed in 45 minutes
- **Efficiency:** Rely on keyboard shortcuts for speed

### Development Approach

- **No overengineering** - Keep solutions simple and practical
- **TDD (Test-Driven Development)** - Not mandatory, but read about it
- **Concurrency knowledge** - Refresh knowledge of concurrent code

## Technical Questions for Developers

### Core Concepts

1. **Serverless** - Understanding serverless architecture
2. **API Gateway** - API gateway patterns and usage
3. **Sentry** - Error tracking and monitoring
4. **Authentication and Authorization** - Security concepts
5. **View vs Materialized View** - Database concepts
6. **PostgreSQL Triggers** - Database triggers in PostgreSQL
7. **Airflow vs Celery** - Task queue and workflow management
8. **CORS** - Cross-Origin Resource Sharing
9. **Port vs Socket** - Network concepts

### Development Practices

10. **CI/CD** - What is CI/CD and why is it needed?
11. **JWT** - JSON Web Tokens
12. **RBAC, ABAC, ACL** - Access control models
13. **CQRS** - Command Query Responsibility Segregation
14. **Code Quality Tools** - ruff/black/isort/mypy/pyright
15. **Environments** - Development, Staging, and Production setup
16. **Alembic** - Database migrations
17. **Transactions** - Database transaction concepts
18. **CAP & BASE & ACID** - Database consistency models
19. **Primary Keys** - int vs uuid comparison

### Architecture and Design

20. **Microservice Patterns** - Microservice architecture patterns
21. **Microservice Chassis** - Microservice framework patterns
22. **ERD Diagram, UML Sequence** - Diagramming and modeling
23. **Async def vs def** - Asynchronous programming in Python
24. **Testing** - What types of tests are used?
25. **RSS Feed** - RSS feed implementation
26. **HTTP Methods** - Which HTTP method is idempotent?
27. **Database Optimization** - How to optimize databases?
28. **Most Difficult Problem** - Tell me about the most difficult problem you worked on

## HR Questions to Ask

### Team and Structure

1. **Reporting Structure** - Who do I report to and what is expected of me? Is the team already formed?
2. **Career Growth** - How and under what circumstances can I advance to Product role?
3. **Documentation** - In what form is documentation stored? Are any standards followed?
4. **Tools** - What tools do system analysts use in their work?
5. **Compensation** - What are the bonuses? How does salary indexing work?

## What Not to Say

### Red Flags to Avoid

- ❌ **"I've never worked anywhere for more than a year"** - Shows lack of commitment
- ❌ **"My questions don't affect anything, I just want to ask"** - Shows lack of engagement

**From interview to master delivery:**
- Avoid statements that show disinterest or lack of commitment
- Show genuine interest in the role and company
- Ask meaningful questions that demonstrate your understanding

## Preparation Tips

1. **Review core concepts** - Refresh knowledge of listed technical topics
2. **Practice coding** - Be comfortable with unit tests and mocks
3. **Prepare questions** - Have meaningful questions ready for HR
4. **Know your story** - Be ready to discuss your most difficult problem
5. **Show engagement** - Demonstrate genuine interest in the role
