/--------------Рекомендательные системы (RecSys)-------------/

-Неперсонализированные РС (неперсенифицированные) # строится на кол-ве просмотров и лайков  
-Контентные РС (персонализированные, строится на том что смотрел человек) # ищутся похожие фильмы по описанию того фильма, который смотрел пользователь  
-Коллаборативная фильтрация:  
    User-based # порекомендуем человеку то, что смотрели люди похожие на него  
    Item-based # порекомендуем к купленному товару сопутствующие товары  
     
Матричные разложения (SVD) 

Оценка качества

РС нужны, когда:  
- пользователь не знает точно что хочет  
- число доступных объектов не позволяют человеку обозреть их все

/---------Рекомендательные системы на базе коллаборативной фильтрации-----------/  
Explicit feedback - явная обратная связь от пользователя (отзыв, лайк, перепост)  
Implicit feedback - неявная обратная связь от пользователя (время проведенное на ресурсе, факт/кол-во просмотра товара)

Зная предпочтения конкретного пользователя и остальных пользователей, выдать этому пользователю рекомендацию

Метод сходства по пользователя м

/---------Матричная факторизация-----------/  
Матричная факторизация счиается стандартом в индустрии рекомендательных систем

SVD - singular value decomposition

ALS - Alternating Least Squares

BMF - boolean matrix factorization      {0,1}

NMF - non-negative matrix factorization {1, ..., 5}

PLSA - [0,1] вероятностная скрытая

PCA - principial component analysis

/-----------SVD-------------/  
Не работает на sparse матрицах.  
Используется для снижения размерности данных, для этого из диагональной матрицы выкидываем малозначимые факторы, затем восстанавливаем матрицу  
Находится в одном ряду с PCA  
Выявить скрытые факторы в данных, которые можно использовать дальше.  
Раскладывает матрицу на 3 матрицы.  
Матрица с Users  
Матрица с Item  
Диагональная матрица

/-----------ALS-------------/

Раскладывает матрицу на 2 матрицы. Нет диагональной матрицы.
